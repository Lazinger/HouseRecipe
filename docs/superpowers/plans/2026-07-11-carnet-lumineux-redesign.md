# Refonte "Carnet lumineux épuré" Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the "carnet-scrapbook" visual system (kraft/coral/teal/amber, torn-paper `clip-path`, washi tape, rotated cards, paper shadows) across the whole site with the approved "carnet lumineux épuré" system (near-white background, emerald-green primary accent, Fraunces-italic + DM Sans typography, thin-line separators, flat surfaces, no rotation/tape/torn edges).

**Architecture:** Pure CSS/HTML/vanilla-JS static site, no build step, no test framework, not a git repository. All visual logic lives in `style.css` (tokens + component rules) and recipe/UI markup is generated by template strings in `script.js`; `index.html` holds the static shell (header, drawer, FAB, toast, view containers). Each task rewrites one coherent section of `style.css`, updates the matching `script.js` template(s) if they reference removed classes (`.tape`, `.detail-hero-fill`, `.detail-doodle`), and is verified by driving the real app in the Browser pane (no automated tests exist in this repo).

**Tech Stack:** Vanilla HTML/CSS/JS, Google Fonts (Fraunces, DM Sans, Caveat), served via the project's `static-serve` dev config (`http://localhost:3000`).

## Global Constraints

- No test framework exists in this repo — every task's verification step is a manual browser check via the Browser pane tools (`preview_start`/`navigate`/`javascript_tool`/`computer` screenshot), not an automated test run.
- This is **not a git repository** (confirmed: `Is a git repository: false`) — there are no "commit" steps in this plan. Each task ends with a save + browser verification checkpoint instead.
- Design tokens, palette, typography, and component principles are fixed by the approved spec: [docs/superpowers/specs/2026-07-11-carnet-lumineux-redesign.md](../specs/2026-07-11-carnet-lumineux-redesign.md). Do not deviate from the hex values or font choices there without checking with the user first.
- Never reintroduce, in new code: `clip-path: var(--torn)` / `var(--torn-bottom)`, `.tape` elements, `transform: rotate(...)` on cards/buttons, or `--shadow-paper`/`--shadow-paper-sm`. These are being removed project-wide.
- The user explicitly wants this treated as one focused pass — once all tasks are verified, design work is set aside; don't add scope beyond what's listed here.
